EVENT CONTRACT SPECIFICATION:

HTTP REST Endpoints:
  POST   /api/earth-engine/analyze              → { task_id, status: "queued", ... }
  GET    /api/earth-engine/tasks/{task_id}      → { status, progress_pct, phase, ... }
  GET    /api/earth-engine/tasks/{task_id}/result → { provenance, geometry, tiles, metrics }
  GET    /api/earth-engine/tiles/{task_id}/{layer}/{z}/{x}/{y}.png → PNG tile

WebSocket Events (NEW):
  Emit:   ee_analysis_start { type, parameters }
  Recv:   ee_analysis_update { type, task_id, phase, progress_pct, message }
  Recv:   ee_analysis_complete { type, task_id, analysis_provenance, geometry, tiles, metrics, map_actions }

Payload Schema Examples:
  SUBMIT:
    { "analysis_type": "flood_extent",
      "parameters": { "baseline_window": ["2025-06-01", "2025-09-30"],
                      "event_window": ["2026-01-01", "2026-02-28"],
                      "dataset": "COPERNICUS/S1_GRD", "threshold_db": -3.0 } }

  RESULT:
    { "analysis_provenance": { /* detailed metadata */ },
      "geometry": { /* GeoJSON FeatureCollection */ },
      "tiles": { "change_detection": { "url": "...", "style": {...} }, ... },
      "metrics": { "flood_area_sqkm": 1.56, "growth_rate_estimate": {...}, ... } }

═══════════════════════════════════════════════════════════════════════════════

KEY IMPLEMENTATION DECISIONS:

├─ Async Pattern: asyncio (non-blocking task polling)
├─ Tile Caching: Hybrid (live from EE + local fallback)
├─ Task TTL: 1 hour (reasonable cleanup window)
├─ Fallback: Static data (zero downtime if EE unavailable)
├─ Auth: Environment variables (GOOGLE_APPLICATION_CREDENTIALS)
├─ Rate Limiting: Exponential backoff (1s, 2s, 4s, 8s, 16s)
└─ Backward Compat: Keep ee_update message for old clients

═══════════════════════════════════════════════════════════════════════════════

RISK MITIGATION:

Risk                        Mitigation
─────────────────────────────────────────────────────────────────────
EE API downtime             → Fallback to static data
Network interruption        → WebSocket reconnect resumes from task_id
Task expiration             → 1-hour TTL cache + restart capability
Auth failure                → Logged warning + static-only mode
Slow tile generation        → Exponential backoff on polling
Backward compatibility      → Keep ee_update event for v1 clients

SUCCESS CRITERIA (POST-IMPLEMENTATION):

Submission Phase:
  ✓ User clicks "Run Live Analysis" button
  ✓ Task submitted to EE, task_id returned
  ✓ ee_analysis_start event emitted

Polling Phase:
  ✓ Backend polls every 5 seconds
  ✓ ee_analysis_update events with progress (0-100%)
  ✓ Phase labels update (baseline_composite, change_detection, etc.)
  ✓ Frontend progress bar animates

Completion Phase:
  ✓ ee_analysis_complete event emitted
  ✓ GeoJSON vectors render on globe
  ✓ Raster tiles display as imagery layers
  ✓ Each tile layer can be toggled independently
  ✓ Provenance shows "LIVE" status + timestamp

Robustness:
  ✓ Static data fallback if EE unavailable
  ✓ WebSocket reconnect resumes from task_id
  ✓ Old clients (v1) continue working
  ✓ Graceful error handling, no 500 errors
